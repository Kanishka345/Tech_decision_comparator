# Kiro Prompts Used

## Prompt 1: Initial Application Scaffolding

Build a professional web-based decision comparison tool.

Requirements:
- Compare two technology options
- Accept user constraints such as cost, scalability, ease of use, and performance
- Display pros, cons, and trade-offs
- Provide a recommendation based on the selected constraint
- Use HTML, CSS, and JavaScript
- Keep the design clean and professional, suitable for a hackathon demo

---

## Prompt 2: UI and UX Enhancement

Improve the UI and layout of the decision comparison tool.

Requirements:
- Modern, professional appearance
- Card-based layout for comparison results
- Clear input section and result section
- Responsive design for different screen sizes

---

## Prompt 3: Trade-off Explanation Logic

Enhance the JavaScript logic to clearly explain trade-offs between the selected options.

Requirements:
- Explain why one option is preferred over the other
- Keep explanations simple and human-readable
- Focus on real-world engineering trade-offs
